How It Works

Mist sits between your agent and dangerous operations

1

Agent triggers a risky action

OpenClaw's approval engine detects when an agent attempts something that needs human sign-off — file deletion, external API calls, privilege changes.

➔
2

Mist sends an MFA challenge

The action is paused. Mist creates a challenge and routes it to you. You get notified on your phone via SafeRoom.

➔
3

You approve or deny

Review the action, tap approve or deny. The decision flows back through Mist to OpenClaw, and the agent continues or stops. Seconds, not minutes.

Why Mist

Human-in-the-loop, not human-in-the-way

Safe operations flow through untouched. Only risky actions pause for approval. Your agents stay fast.

Privacy by design

All sensitive data is hashed and encrypted on your device before it ever leaves your phone. No raw data is transmitted.

Three lines of config

Install the plugin, scan a QR code, restart. Mist works with your existing OpenClaw setup — no new infrastructure.

Cryptographic device auth

Ed25519 signatures bind each machine to your identity. Scoped access tokens. Challenge-response replay protection.

Get Started

Up and running in a few minutes

1

Install the plugin

bash 
openclaw plugins install @percent-20/openclaw-mfa-gate
2

Pair with SafeRoom

Run the setup wizard. It generates a QR code — scan it with the SafeRoom app to bind your vault.

bash 
openclaw mfa
📱
SafeRoom is in early access

The companion app isn't on stores yet. Join the waiting list to get notified.

3

Configure and restart

Tell OpenClaw to route approvals through Mist:

~/.openclaw/config.json 
{
  "approvals": {
    "provider": "mist"
  },
  "channels": {
    "mist": {
      "enabled": true
    }
  }
}
bash 
openclaw restart

That's it. Risky agent actions will now require your approval via SafeRoom.

Privacy First

Your data never leaves your device in raw form

Hashed and encrypted on-device

SafeRoom hashes and encrypts all sensitive data locally on your phone before anything is transmitted. The server never sees raw coordinates, identifiers, or personal data.

Device-bound keys

Mist generates Ed25519 key pairs that stay on your machine. Private keys are never transmitted. Each device has its own cryptographic identity.

No third-party tracking

No analytics, no ad networks, no data sharing. The only communication is between your devices and the Mist service.

FAQ

What kinds of actions trigger MFA?

That's up to your OpenClaw configuration. Typically: file deletion, external API calls, privilege escalation, bulk operations, and anything else you define as risky. Safe, read-only operations pass through without interruption.

Is my location or personal data sent to a server?

No raw data ever leaves your phone. SafeRoom hashes and encrypts everything locally on-device before transmitting. The server works with cryptographic proofs, not your actual data.

What happens if the server is unreachable?

Mist fails closed. If the challenge can't be created or the response times out, the action is denied by default. You can configure the timeout window in your OpenClaw settings.

Can I use Mist on multiple machines?

Yes. Each machine generates its own Ed25519 key pair and binds to your vault. Run openclaw mfa on each machine and scan the QR code with the same SafeRoom account.

What if QR scanning doesn't work?

You can manually bind using the key displayed in SafeRoom: openclaw mfa bind mfa:bind:<vault_uid>:<token>
