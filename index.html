The solution

Three steps between your agent and disaster

Mist intercepts risky actions and checks if you're present before executing

01

Agent hits a guardrail

OpenClaw detects a risky action: file deletion, API call, privilege change. The execution is paused.

02

Your phone verifies presence

SafeRoom checks your environment: WiFi, time, location. All encrypted on-device. The server sees only proofs.

03

Approve or deny in milliseconds

Safe environment? Approved instantly. Not safe? You get a push notification to review. No context switches, no slowdowns.

Get started

Up and running in five minutes

1

Install the plugin

bash 
openclaw plugins install @percent-20/openclaw-mfa-gate
2

Pair with SafeRoom and define your safe environment

Run the setup wizard. It generates a QR code. Scan it with the SafeRoom app to bind your vault.

bash 
openclaw mfa

Once paired, SafeRoom walks you through creating a safe environment: the bubble that defines when your agent can act without asking.

A Set security level

Choose how strict. "Time & Place" auto-approves at the right location during work hours. "Maximum Security" adds device proximity checks.

B Select environment signals

Combine signals: location, WiFi, time of day, nearby devices, NFC tags. All data is hashed and encrypted before leaving your phone.

When all your conditions match, risky agent actions are approved silently. When they don't, SafeRoom sends you a push notification to review.

You can define multiple environments (home, office, café) with different rules and time windows.

📱
SafeRoom is in early access

3

Configure and restart

Tell OpenClaw to route approvals through Mist:

~/.openclaw/config.json 
{
  "approvals": {
    "provider": "mist"
  },
  "channels": {
    "mist": {
      "enabled": true
    }
  }
}
bash 
openclaw restart

That's it. Every risky agent action now requires your approval via SafeRoom.

FAQ

Common questions

What kinds of actions trigger MFA?

That's up to your OpenClaw configuration. Typically: file deletion, external API calls, privilege escalation, bulk operations, and anything you define as risky. Safe, read-only operations pass through.

Is my location or personal data sent to a server?

No raw data ever leaves your phone. SafeRoom hashes and encrypts everything locally on-device before transmitting. The server works with cryptographic proofs, not your actual data.

What happens if the server is unreachable?

Mist fails closed. If the challenge can't be created or the response times out, the action is denied by default. You can configure the timeout in your OpenClaw settings.

Can I use Mist on multiple machines?

Yes. Each machine generates its own Ed25519 key pair and binds to your vault. Run openclaw mfa on each machine and scan the QR code with the same SafeRoom account.

What if QR scanning doesn't work?

You can manually bind using the key displayed in SafeRoom: openclaw mfa bind mfa:bind:<vault_uid>:<token>
