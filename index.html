How It Works

Three components, one seamless flow

1
⚠

OpenClaw Detects Risk

When an AI agent attempts a risky action (file deletion, data export, privilege escalation), OpenClaw's approval engine flags it and pauses execution.

➔
2
◉

AirVaults Checks Location

The Mist channel sends a challenge to AirVaults. The server checks whether the vault owner is physically inside the vault's geofenced area using real-time location tracking.

➔
3
✓

Auto Approve or Deny

If you're inside the vault: approved instantly. If not: denied. No pop-ups, no push notifications, no manual taps required. The decision happens in milliseconds.

Installation

Get up and running in 6 steps

1

Install OpenClaw

If you don't have OpenClaw installed yet, install it globally:

bash 
npm install -g @percent-20/openclaw

Verify the installation:

bash 
openclaw --version
2

Install the Mist Channel Plugin

Install the Mist (AirVaults MFA) plugin for OpenClaw:

bash 
openclaw plugins install @percent-20/openclaw-mfa-gate

This registers the mist channel with your OpenClaw gateway. You can verify it's loaded:

bash 
openclaw channels list
# Output:
# ✓ mist    Mist (AirVaults MFA)    v1.0.0
3

Get the SafeRoom Mobile App

SafeRoom is the mobile companion app that manages your vault presence. It tracks your location and creates the geofenced vaults that Mist uses for approvals.

📱
Early Access

SafeRoom is currently in early access and not yet available on app stores. Join the waiting list below to get notified when it launches.

Join Waiting List
4

Run the MFA Setup Wizard

The setup wizard generates a QR code that you scan with SafeRoom to bind your vault to this OpenClaw instance:

bash 
openclaw mfa

This will:

  1. Generate a unique instance ID for your OpenClaw installation
  2. Display a QR code in your terminal (and optionally open it in your browser)
  3. Wait up to 5 minutes for you to scan the QR code with SafeRoom
  4. Once scanned, SafeRoom creates an MFA vault and binds it to your instance
  5. The wizard auto-detects the binding and saves the vault UID and access token

If QR scanning fails, you can manually bind using the key displayed in SafeRoom:

bash 
openclaw mfa bind mfa:bind:<vault_uid>:<token>
5

Configure Exec Approvals

Edit your OpenClaw configuration to route approval requests through the Mist channel. Open ~/.openclaw/config.json:

~/.openclaw/config.json 
{
  "approvals": {
    "provider": "mist",
    "riskThreshold": "medium"
  },
  "channels": {
    "mist": {
      "enabled": true,
      "apiBaseUrl": "https://tryme.abraquadabra.com"
    }
  }
}

The riskThreshold determines which actions require MFA approval. Set it to "low" for maximum security or "high" for only the most dangerous operations.

6

Restart and Test

Restart the OpenClaw gateway to pick up the new configuration:

bash 
openclaw restart

Now test with a risky action. Make sure you're physically inside your vault location first:

bash 
openclaw exec "delete temp files in /tmp/staging"

# Expected output:
# ▸ Risk detected: file_deletion, destructive_action
# ▸ MFA challenge sent to Mist channel...
# ▸ Checking vault presence...
# ✓ Approved (owner inside vault). Executing...

Check the MFA status to see the challenge log:

bash 
openclaw mfa status

# MFA Vault Status
# ────────────────
# Vault:       My Workspace Vault
# Challenges:  14 total (12 approved, 2 denied)

Risk Levels

OpenClaw assigns risk levels to actions. The riskThreshold in your config determines which levels trigger MFA:

Low 0.2

Read operations, listing files, non-destructive queries

Medium 0.5

File modifications, configuration changes, data updates

High 0.8

File deletion, bulk operations, data exports

Critical 1.0

Privilege escalation, recursive deletion, system directory access

How Location-Based Approval Works

The Vault Presence Model

AirVaults uses a geofenced vault model. When you create an MFA vault in SafeRoom, you define a physical location and radius. The vault exists at that spot in the real world.

The SafeRoom app continuously reports your GPS coordinates to AirVaults. A Visit Tracker backed by Redis maintains real-time state of whether you are inside or outside each vault's boundary.

When a challenge arrives, the server simply checks: "Is the vault owner currently inside this vault?" If yes, the action is approved. If no, it's denied. The decision is made server-side in milliseconds — no round-trip to your phone needed.

Why This Works

  • Zero friction — No pop-ups, no codes to type, no biometrics to scan
  • Physical security — An attacker would need to be at your exact location
  • Instant decisions — Sub-second approval latency
  • Automatic — Walk into your office, approvals just work. Walk away, they stop

Mobile Override

Even after an automatic decision, you can override it from SafeRoom. The app shows pending challenges and lets you manually approve or deny. This is useful if you step outside briefly or want to block a specific action despite being in the vault.

Security

Ed25519 device authentication for gateway access

The Mist channel uses Ed25519 cryptographic signatures to authenticate with the OpenClaw gateway. During setup, a key pair is generated and stored locally:

~/.openclaw/identity/
~/.openclaw/identity/
├── device.json          # Ed25519 key pair + device ID
└── device-auth.json     # Scoped access tokens

When the Mist channel connects to the gateway, it signs a payload containing its device ID, requested scopes, and a timestamp. The gateway verifies the signature and grants the operator.approvals scope, which allows the channel to resolve approval requests.

If the gateway requires a challenge-response (protocol v3), the channel receives a nonce, re-signs with the nonce included, and sends back the proof. This prevents replay attacks.

FAQ & Troubleshooting

What happens if I'm not in the vault when an action triggers?

The challenge is automatically denied. The AI agent receives a "deny" decision and the action is blocked. You can also monitor denied challenges in openclaw mfa status.

Can I approve an action from my phone even if I'm outside the vault?

Yes. SafeRoom shows pending challenges and lets you manually approve or deny regardless of your location. The automatic location check happens first, but your manual response can override it.

How accurate is the location check?

It depends on your phone's GPS accuracy and the vault radius you configure. AirVaults uses a geohash-based system with multiple precision levels. For typical office or home use, a 50-100m radius works well.

What if the AirVaults server is unreachable?

If the Mist channel cannot reach the AirVaults API, the challenge creation fails and the approval times out. OpenClaw will deny the action by default (fail-closed). You can configure the timeout via pollTimeoutMs.

Does this work with multiple machines?

Yes. Each machine gets its own device identity (Ed25519 key pair) and binds to the same MFA vault. Run openclaw mfa on each machine and scan the QR code with the same SafeRoom account.

The setup wizard times out before I can scan the QR code

The wizard waits 5 minutes. If it times out, just run openclaw mfa again. You can also use the manual bind command: openclaw mfa bind mfa:bind:<vault_uid>:<token> with the key shown in SafeRoom.

I see "missing operator.approvals scope" errors

This means the device auth handshake failed. Check that ~/.openclaw/identity/device.json exists and contains valid keys. You can regenerate by deleting the identity files and running openclaw mfa again.

How do I check my vault status?

Run openclaw mfa status to see your vault name, total challenges, approval/denial counts, and recent challenge history.

Is my location data sent to a third party?

Your GPS coordinates are sent only to the AirVaults server. The server uses them to determine vault presence and does not share them with any third party. Location data is encrypted in transit via HTTPS.
